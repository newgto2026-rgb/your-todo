# 요약
- 상단 프로필 아이콘을 누르면 화면 오른쪽에서 프로필 메뉴가 슬라이드 인된다.
- MVP는 서버 변경 없이 현재 세션의 닉네임과 이메일을 읽기 전용으로 보여준다.
- 닉네임 변경은 계정 중복 검증, 친구 검색 인덱스, 기존 할당/알림 표시명 갱신 등 서버 영향이 있어 이번 범위에서 제외한다.
- 메뉴에는 로그아웃, 닉네임 복사, 알림/앱 설정 진입, 정책/버전 정보처럼 일반 앱에서 기대되는 최소 계정 메뉴 요소를 제공한다.

# 배경
- 현재 Todo, Calendar, Friends 화면은 프로필 initial을 헤더에 표시하지만, 사용자가 프로필을 눌렀을 때 수행할 수 있는 계정 액션이 없다.
- 앱은 로그인/닉네임 온보딩과 `SignOutUseCase`를 이미 가지고 있으므로 MVP 로그아웃은 신규 서버 API 없이 구현 가능하다.
- 친구 기능에서는 닉네임이 검색 키로 쓰이므로, 사용자가 내 닉네임을 확인하고 복사할 수 있는 진입이 있으면 친구 추가 흐름을 보조할 수 있다.

# 제품 목표
- 사용자가 어느 주요 탭에서든 프로필 아이콘을 눌러 계정 정보를 확인한다.
- 사용자가 닉네임을 변경하지 않고도 친구에게 공유할 닉네임을 쉽게 복사한다.
- 사용자가 명확한 확인 절차를 거쳐 안전하게 로그아웃한다.
- 계정 관련 보조 진입점을 한곳에 모아 앱의 완성도를 높인다.

# 대상 사용자
- Google 로그인 후 Todo, Calendar, Friends 기능을 사용하는 일반 사용자.
- 친구 추가를 위해 자신의 닉네임을 공유해야 하는 사용자.
- 공용 기기나 테스트 계정에서 로그아웃이 필요한 사용자.

# MVP 추천 메뉴 구성
- 프로필 요약:
  - 원형 아바타 initial
  - 닉네임
  - 이메일
  - 닉네임은 읽기 전용이며 변경 버튼을 노출하지 않는다.
- 빠른 액션:
  - `Copy nickname`: 현재 닉네임을 클립보드에 복사한다.
  - `Notification settings`: Android 앱 알림 설정 또는 앱 설정으로 이동한다.
- 정보 섹션:
  - `Privacy Policy`
  - `Terms of Service`
  - 앱 버전
- 계정 액션:
  - `Log out`
  - 로그아웃은 destructive 스타일과 확인 다이얼로그를 사용한다.

# 핵심 사용자 시나리오
1. 사용자는 Todo, Calendar, Friends 화면 상단의 프로필 아이콘을 누른다.
2. 오른쪽에서 메뉴가 열리고 사용자는 닉네임, 이메일, 앱 정보를 확인한다.
3. 사용자는 `Copy nickname`을 눌러 닉네임을 복사하고 친구에게 공유한다.
4. 사용자는 `Log out`을 누른 뒤 확인 다이얼로그에서 다시 확인한다.
5. 로그아웃이 성공하면 세션이 정리되고 로그인 화면으로 이동한다.

# 기능 요구사항
- 모든 주요 탭의 프로필 아이콘은 클릭 가능한 버튼이어야 한다.
- 프로필 버튼은 접근성 content description과 최소 터치 영역을 가진다.
- 프로필 버튼을 누르면 오른쪽 profile menu drawer가 열린다.
- drawer는 scrim, 외부 탭 닫기, back 닫기, drag close를 지원한다.
- drawer는 현재 `AuthSession.user.nickname`, `AuthSession.user.email`을 표시한다.
- 닉네임이 없거나 공백이면 fallback 문구를 표시하고 복사 액션은 비활성화한다.
- 닉네임 복사 성공 시 snackbar 또는 toast로 완료 피드백을 제공한다.
- 로그아웃 버튼을 누르면 확인 다이얼로그를 표시한다.
- 로그아웃 진행 중에는 로그아웃 버튼을 비활성화하고 progress 상태를 표시한다.
- 로그아웃 성공 시 drawer를 닫고 인증 게이트가 로그인 화면을 표시한다.
- 로그아웃은 기존 `SignOutUseCase`를 사용해 push token 삭제와 세션 삭제를 함께 수행한다.
- 사용자 노출 문자열은 `values`, `values-ko` 리소스로 관리한다.

# 화면 요구사항
- drawer 위치:
  - 오른쪽 edge에서 slide in/out된다.
  - 너비는 compact 화면에서 화면 너비의 대부분을 사용하되, large 화면에서는 최대 너비를 제한한다.
  - 상태바/내비게이션바 inset을 침범하지 않는다.
- drawer 상단:
  - 닫기 아이콘 버튼
  - profile initial avatar
  - 닉네임
  - 이메일
- 메뉴 항목:
  - 아이콘 + 라벨 + 필요한 경우 보조 텍스트를 사용한다.
  - destructive 액션인 로그아웃은 하단에 분리 배치한다.
- 로그아웃 확인:
  - 제목, 설명, 취소, 로그아웃 버튼을 가진다.
  - 확인 전에는 세션을 삭제하지 않는다.

# 상태 정의
- `Closed`: 메뉴가 닫힌 기본 상태.
- `Open`: 세션 정보를 표시하며 메뉴가 열린 상태.
- `CopySuccess`: 닉네임 복사 완료 피드백이 필요한 상태.
- `LogoutConfirming`: 로그아웃 확인 다이얼로그가 열린 상태.
- `LogoutInProgress`: 로그아웃 처리 중 상태.
- `LogoutFailed`: 로그아웃 실패 메시지를 표시하는 상태.

# 예외 케이스
- 세션이 없는 상태에서 프로필 메뉴를 열려고 하면 메뉴를 열지 않고 인증 게이트 상태를 유지한다.
- 이메일이 비어 있으면 이메일 영역을 숨기거나 `Email unavailable` fallback을 사용한다.
- 닉네임이 비어 있으면 복사 액션을 비활성화하고 닉네임 설정이 필요하다는 읽기 전용 안내만 표시한다.
- 알림 설정 intent를 열 수 없으면 앱 상세 설정 화면으로 fallback한다.
- 로그아웃 실패 시 drawer를 유지하고 retry 가능한 오류 메시지를 표시한다.

# 정책
- 이번 MVP에서는 닉네임 변경을 제공하지 않는다.
- 이번 MVP에서는 계정 삭제, 프로필 이미지 업로드, 로그인 제공자 연결/해제를 제공하지 않는다.
- 신규 서버 API, DB 마이그레이션, DataStore 스키마 변경은 하지 않는다.
- `core:* -> feature:*` 의존은 추가하지 않는다.
- 앱 셸은 drawer 컨테이너와 전역 계정 액션만 조율하고, feature 구현 상세를 직접 참조하지 않는다.
- 각 feature 화면은 프로필 클릭 이벤트를 상위로 전달하고, drawer 상태와 로그아웃 절차를 소유하지 않는다.

# API / 데이터 요구사항
- 서버 API 변경 없음.
- `core:network` 변경 없음.
- 기존 `AuthSession.user.nickname`과 `AuthSession.user.email`을 사용한다.
- 로그아웃은 기존 `SignOutUseCase`를 사용한다.
- 앱 버전은 BuildConfig 또는 패키지 정보에서 읽는다.
- Privacy Policy / Terms URL은 빌드 설정 또는 리소스 상수로 관리한다.
