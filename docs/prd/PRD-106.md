# 요약
- 현재 앱은 로컬 우선 Todo/Calendar MVP다.
- 다음 기능은 신규 서버 기능보다 오늘 실행과 재계획을 강화하는 `Today Planner / 빠른 재계획`으로 좁힌다.
- Calendar 날짜에서 새 Todo를 추가할 때 선택 날짜를 due date로 미리 채워, Calendar와 Todo 흐름을 연결한다.

# 가정
- 대상 사용자는 개인 할 일 관리 사용자다.
- 앱은 당분간 로컬 전용으로 유지한다.
- 다음 PR은 한 번에 리뷰 가능한 MVP 범위여야 한다.
- 분석 이벤트를 추가하더라도 Todo 제목, 메모, 정확한 날짜/시간, 알림 시각은 수집하지 않는다.

# 제품 목표
- 사용자가 Today 탭에서 오늘 처리할 일과 밀린 일을 빠르게 판단한다.
- 사용자가 못 할 일을 목록에서 바로 내일로 미루거나 날짜를 제거한다.
- 사용자가 Calendar에서 선택한 날짜에 바로 할 일을 추가한다.

# 대상 사용자
- 개인 일정과 할 일을 로컬 앱에서 관리하는 사용자.
- 하루 단위로 할 일을 확인하고, 밀린 일을 다음 날로 재계획하는 사용자.

# 핵심 사용자 시나리오
1. 사용자는 Today 탭을 열어 기한 지난 일, 시간이 지정된 오늘 일, 오늘 일, 높은 중요도 일을 확인한다.
2. 사용자는 항목을 완료하거나, 내일로 미루거나, 날짜를 제거한다.
3. 사용자는 Calendar에서 특정 날짜를 선택하고 해당 날짜가 미리 채워진 새 Todo를 추가한다.

# 기능 요구사항
- Today 탭은 미완료 항목을 실행 중심으로 분류한다.
- Today 탭 분류는 중복 노출을 피한다.
- 빠른 액션은 완료, 내일로 미루기, 날짜 제거를 제공한다.
- 빠른 액션 후에는 Undo 가능한 snackbar를 제공한다.
- Calendar agenda는 선택 날짜에 새 Todo를 추가하는 CTA를 제공한다.
- Calendar에서 시작한 Todo 추가 화면은 선택 날짜를 due date로 prefill한다.
- 저장 후 Calendar agenda는 해당 날짜의 새 Todo를 표시한다.

# 화면 요구사항
- Today 탭 섹션:
  - `Overdue`: 오늘 이전 due date이고 미완료
  - `Timed today`: 오늘 due date와 due time이 있고 미완료
  - `Today`: 오늘 due date이고 due time이 없는 미완료
  - `High priority`: due date가 오늘/과거가 아니고 중요도가 HIGH인 미완료
- Todo 편집 시 신규는 `New Task`, 수정은 `Edit Task` 제목을 사용한다.
- Calendar agenda에는 `Add task for this date` CTA를 노출한다.
- 사용자 노출 문자열은 `values`, `values-ko` 리소스로 관리한다.

# 상태 정의
- `Loading`: 데이터 로딩 중.
- `Empty Today`: Today Planner에 표시할 항목 없음.
- `Filtered Empty`: 필터 조건 때문에 결과 없음.
- `Error`: 저장/삭제/재계획 실패.
- `Pending Undo`: 삭제/재계획 직후 복구 가능 상태.

# 예외 케이스
- 빠른 재계획 실패 시 기존 Todo 상태를 유지하고 오류 snackbar를 표시한다.
- 날짜 제거 시 due date, due time, reminder 상태를 함께 정리한다.
- Calendar 추가를 취소하면 선택 날짜와 현재 월은 유지한다.
- 내일로 미루기 시 리마인더가 있으면 due time과 lead minutes 기준으로 재계산한다.
- 재계산된 리마인더 시간이 과거이면 리마인더를 비활성화한다.

# 정책
- 이번 MVP는 로컬 Room/DataStore 기반으로 제한한다.
- 계정, 동기화, 협업, AI 추천, 복잡한 반복 규칙, 카테고리 관리 UI는 제외한다.
- `core:* -> feature:*` 의존은 추가하지 않는다.
- `feature:*:api`는 route 계약만, `feature:*:impl`은 구현 상세만 담당한다.

# API / 데이터 요구사항
- 서버 API 변경 없음.
- `core:network` 변경 없음.
- 기존 Todo 필드인 `title`, `isDone`, `priority`, `dueDate`, `dueTimeMinutes`, reminder 필드를 재사용한다.
- 서버 동기화가 필요해지는 경우 별도 `Sync/API v1` PRD로 분리한다.

- 수집 금지:

# 수용 기준
- [ ] Today 탭에서 기한 지난 항목, 시간이 지정된 오늘 항목, 오늘 항목, 높은 중요도 항목이 섹션으로 표시된다.
- [ ] Today 탭에서 같은 Todo가 여러 섹션에 중복 표시되지 않는다.
- [ ] 내일로 미루기를 실행하면 due date가 내일로 저장되고 Undo가 가능하다.
- [ ] 날짜 제거를 실행하면 due date, due time, reminder 상태가 정리되고 Undo가 가능하다.
- [ ] Calendar에서 특정 날짜를 선택한 뒤 새 Todo 추가를 시작하면 due date가 선택 날짜로 채워진다.
- [ ] Calendar에서 시작한 Todo 추가를 취소해도 선택 날짜와 현재 월이 유지된다.
- [ ] 기존 Todo 추가/수정/삭제/완료/리마인더 흐름이 회귀하지 않는다.

# MVP 범위
- Today Planner 섹션화.
- 내일로 미루기와 날짜 제거 빠른 액션.
- 삭제/재계획 Undo.
- Calendar 날짜 기반 Todo 추가 진입.
- ViewModel 단위 테스트와 Calendar side effect 테스트.
